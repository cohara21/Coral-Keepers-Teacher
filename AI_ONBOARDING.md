# AI Onboarding (Cross-Tool)

Use this file when working in any AI coding tool (Copilot, Cursor, Claude, etc.).

## Read These First (In Order)
1. .github/copilot-instructions.md
2. PROJECT_CONTEXT.md
3. DESIGN_GUIDELINES.md

## Required Working Rules
- Keep Student and Teacher versions aligned when a change applies to both.
- Keep version-specific changes only in the matching folder.
- Preserve existing relative links and asset paths.
- Keep Teacher navigation/routes in Teacher pages and Student navigation/routes in Student pages.
- Reuse existing visual style (typography, spacing, color language) unless explicitly asked to redesign.

## Cory Chat Consistency Contract
When editing Cory chat, keep behavior consistent across pages:
- Trigger at bottom-right.
- Open/close via trigger, close button, outside click, and Escape.
- Empty initial chat body.
- User bubbles right (blue), Cory bubbles left (light gray).
- Input submits with Enter and Send button.

## Before Finishing Any Task
- Verify mirrored Student/Teacher pages were updated where applicable.
- Verify navigation links still point to correct view pages.
- Verify Cory behavior on touched pages.
- Run a syntax/error check on edited files.
- Do not commit .DS_Store.

## Prompt Snippet You Can Paste Into Any Tool
"Before making changes, read .github/copilot-instructions.md, PROJECT_CONTEXT.md, and DESIGN_GUIDELINES.md. Follow those as required project rules for all edits."
